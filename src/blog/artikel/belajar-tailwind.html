Belajar CSS dari Nol: Fundamental CSS untuk Pemula.

Sebelum ngomongin layouting, Flexbox, atau Grid, ada satu hal yang sering disepelekan: fundamental CSS. Dasar yang kuat adalah kunci web yang siap deploy.

Di tulisan ini, kita fokus ke konsep dasar CSS, lengkap dengan istilah dan pengertian yang emang dipakai secara resmi, tapi dijelasin dengan bahasa simpel.

Pengertian CSS

CSS adalah singkatan dari Cascading Style Sheets. Menurut World Wide Web Consortium (W3C), CSS merupakan bahasa stylesheet yang digunakan untuk mendeskripsikan bagaimana elemen-elemen HTML ditampilkan di layar.

Kalau HTML berfungsi sebagai struktur atau kerangka konten, maka CSS berperan sebagai pengatur estetika, mulai dari warna, ukuran teks, jarak antar elemen, hingga tata letak halaman. Tanpa CSS, halaman web tetap bisa diakses, tetapi tampilannya sangat minimal dan kurang nyaman dilihat.

Struktur Dasar

CSS ditulis menggunakan pola dasar yang konsisten, yaitu selector, property, dan value. Struktur ini menjadi fondasi dari seluruh aturan CSS yang ada. 


Selector {
  Property: Value;
}

Struktur ini menunjukkan bahwa CSS bekerja dengan cara memilih elemen tertentu, lalu menerapkan aturan gaya ke elemen tersebut. Konsep ini penting dipahami sejak awal karena semua penulisan CSS, baik yang sederhana maupun kompleks, selalu mengikuti pola ini.

Selector dalam CSS

Selector adalah bagian yang menentukan elemen mana yang bakal dipoles. Menurut dokumentasi MDN Web Docs, selector digunakan untuk “menargetkan elemen HTML berdasarkan tipe, class, id, atribut, atau hubungan antar elemen”. Jenis yang wajib lo tau:

p

Semua Tag P

.box

Class (Reusable)

#header

ID (Unique)

Contoh Penulisan :


p {
  color: red;
}

Property dan Value

Setelah elemen target ditentukan oleh selector, CSS menggunakan property dan value untuk menentukan gaya yang akan diterapkan. Property menunjukkan aspek apa yang ingin diubah, sedangkan value menunjukkan nilai dari perubahan tersebut.

Contoh penulisan property dan value: 


p {
  color: red;
  font-size: 16px;
}

Artinya :

  • color → warna teks
  • font-size → ukuran teks
  • blue & 16px → warna dan ukuran teks (value)

Sebagai contoh, property color digunakan untuk mengatur warna teks, sementara value-nya bisa berupa nama warna, kode hexadecimal, atau format warna lainnya. Hubungan antara property dan value ini menjadi inti dari cara kerja CSS dalam mengatur tampilan elemen.

Pemahaman selector sangat krusial karena kesalahan dalam memilih selector sering kali membuat CSS tidak bekerja sebagaimana mestinya, meskipun penulisannya sudah benar.

Class dan ID dalam CSS

Dalam HTML dan CSS, class dan id digunakan sebagai penanda elemen agar bisa dipilih secara spesifik. Menurut spesifikasi HTML dan CSS, class bersifat tidak unik dan dapat digunakan oleh banyak elemen, sedangkan id bersifat unik dan hanya boleh digunakan satu kali dalam satu halaman.

Class

Dipakai buat:

  • Memilih banyak elemen dengan tampilan yang sama
  • Mengaplikasikan gaya yang konsisten di beberapa tempat

HTML :


<div class="card"></div>

CSS :


.card {
  background-color:white;
}

ID

Dipakai buat:

  • Memilih elemen spesifik yang unik di halaman
  • Menerapkan styling khusus ke satu elemen saja

HTML :


<div id="navbar"></div>

CSS :


#navbar {
  height: 60px;
}

Class umumnya digunakan untuk kebutuhan styling yang berulang, seperti kartu, tombol, atau komponen tertentu. Sementara itu, id lebih sering digunakan untuk elemen yang bersifat spesifik, seperti header utama atau container utama halaman.

Pemahaman perbedaan class dan id penting untuk menjaga struktur CSS tetap rapi dan mudah dikembangkan.

Warna dalam CSS

CSS menyediakan beberapa cara untuk mendefinisikan warna. Warna dapat ditulis menggunakan nama warna, format hexadecimal, maupun format RGB. Menurut MDN, penggunaan format warna seperti hexadecimal dan RGB memberikan fleksibilitas lebih tinggi dalam desain antarmuka.

  • Nama warna → red, blue
  • Hex → #FF0000, #0000FF
  • RGB → rgb(255, 0, 0), rgb(0, 0, 255)

Contoh :


color: red;       /* Nama Warna */
background-color: #00FF00; /* Hexadecimal */

Dalam praktik modern, banyak developer menggunakan warna dengan nilai yang mendekati abu-abu gelap untuk teks dan warna terang untuk latar belakang, karena lebih nyaman dibaca dan terlihat lebih profesional.

Satuan Ukuran dalam CSS

CSS menggunakan berbagai satuan ukuran untuk menentukan dimensi elemen. Satuan seperti pixel (px) bersifat absolut, sedangkan persentase, viewport height (vh), dan viewport width (vw) bersifat relatif. Menurut W3C, penggunaan satuan relatif sering kali lebih disarankan dalam desain responsif karena memungkinkan elemen menyesuaikan diri dengan ukuran layar yang berbeda.

  • px → pixel (paling aman buat pemula)
  • % → persentase
  • vh → viewport height
  • vw → viewport width

Contoh :


width: 100px;   /* Pixel */
height: 50vh;   /* Viewport Height */

Bagi pemula, penggunaan pixel sering disarankan karena lebih mudah dipahami dan memberikan hasil yang konsisten. Setelah konsep dasar dikuasai, barulah satuan relatif digunakan untuk membuat tampilan yang responsif terhadap berbagai ukuran layar.

Margin dan Padding

Margin dan padding merupakan konsep dasar dalam CSS yang berkaitan dengan jarak. Padding adalah jarak antara konten dengan batas elemen, sedangkan margin adalah jarak antara satu elemen dengan elemen lainnya.

Margin

Jarak luar elemen (interaksi antar tetangga)

MARGIN AREA
Elemen Kamu
Padding

Jarak dalam elemen (konten ke dinding)

PADDING AREA
Konten 

.box {
  padding: 20px;
  margin: 15px;
}

Menurut konsep box model CSS, pengaturan margin dan padding sangat memengaruhi tata letak halaman. Kesalahan dalam memahami dua properti ini sering kali menyebabkan layout terlihat tidak rapi atau tidak sesuai harapan.

Box Model CSS

Box model adalah konsep fundamental yang menjelaskan bahwa setiap elemen HTML direpresentasikan sebagai sebuah kotak. Kotak ini terdiri dari content, padding, border, dan margin. Konsep ini dijelaskan secara resmi dalam spesifikasi CSS oleh W3C.

Memahami box model membantu developer memahami mengapa ukuran elemen bisa berubah meskipun nilai width dan height sudah ditentukan. Oleh karena itu, box model menjadi dasar penting sebelum mempelajari layouting lanjutan.

Penutup

Fundamental CSS mungkin terlihat sederhana dan kurang “wah”, tetapi justru di sinilah letak kekuatan utama seorang frontend developer. Dengan memahami konsep dasar seperti selector, property, box model, dan satuan ukuran, proses belajar layouting akan terasa jauh lebih masuk akal.

CSS bukan tentang menghafal, melainkan tentang memahami cara browser menerjemahkan aturan visual. Kalau fondasinya kuat, level lanjutan tinggal soal latihan dan jam terbang.
